SelfWeb3

A web3 solution that strengthens the security of on-chain data ownership, provides privatization guarantees for all on-chain data through mandatory dynamic authorization, instead of wallets, webAuthn + TOTP + zero-knowledge proof

Launch App

Dynamic Authorization

Realize private dynamic authorization through webAuthn and TOTP, support session authentication and secondary authorization of sensitive operations, etc., provide web2 environment security for private web3 data, rely on wallet public and private keys, only to ensure the security of A to B, but not including whether A and B are legal, the key to data ownership should be yourself, not the wallet

The selfWeb3 contract side manages private data for users in the form of permission verification for all sensitive operations, including tokens, NFT and user data on the chain, etc., completely controls the ownership of data on the chain from the source, must first complete the private dynamic authorization off the chain, and then generate zero-knowledge proofs, and finally verify and confirm the rights inside the contract

Open source

All source code of selfWeb3 is open source on Github, including contracts, front-end and back-end, and related service components. Since the authorization operation is completed off-chain, if you are worried about the security of private dynamic authorization, you can build and deploy it yourself, but only for your own use. Please do not use any unofficial public sites for interaction
